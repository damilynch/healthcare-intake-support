My Standards

The backbone behind every engagement

These are non-negotiable clinical registration standards — not suggestions. They are what separates precise revenue cycle administration from generic data entry.

🪪 The 6-Identifier Verification Standard

Every registration must capture and verify all six identifiers — no exceptions, no shortcuts.

Full legal name (exactly as on Gov ID — no nicknames)
Date of birth — MM/DD/YYYY with 4-digit year
Government-issued ID number (not SSN as primary)
Insurance policy number AND group number
Current residential address (no PO boxes)
Primary phone number with area code

📋 The 7-Step Registration Workflow

Same steps, same order, every patient, every visit. Consistency eliminates variability.

Collect documents before touching the computer
Visual verification — photo match + document validity
Data entry with source document visible at all times
Read-back all numerical fields against document
Complete insurance data character by character
Inline quality check before saving the record
Verbal confirmation of address + phone with patient

🔒 The 5-Step Insurance Verification Standard

Real-time verification — not visual card inspection. Coverage confirmed before services.

Collect current card — ask "Is this for today's visit?"
Access payer portal, integrated PM system, or clearinghouse
Enter patient data exactly as shown on insurance card
Review: status, dates, plan type, copay, network, auths
Document confirmation number + communicate to patient

📐 Mandatory Field Standard (CMS 1500)

Every mandatory field complete before the patient receives clinical services.

Core demographics: name, DOB, sex at birth, SSN
Contact: residential address (no PO boxes), phone + area code
Insurance: carrier, member ID, group number, effective dates
Dependent coverage: policyholder name + DOB + relationship
Benchmark: error rate maintained below 5%
Target: 95%+ clean claim rate on first submission
